# Gemini 3.1 Flash Live Upgrade

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Support both Gemini 2.5 native-audio and 3.1 Flash Live models, controlled by the model string constant in server.py. Default to 3.1.

**Architecture:** The server defines `GEMINI_LIVE_MODEL` and serves it to the Kotlin app via `/api/config`. The Kotlin `GeminiLiveClient` checks `model == "gemini-3.1-flash-live-preview"` to branch protocol behavior (WebSocket endpoint, thinking config, text sending method). Python test harness gets the same treatment.

**Tech Stack:** Python (FastAPI), Kotlin (OkHttp WebSocket), google-genai SDK

```python
@pytest.mark.voice
class TestGemini31Smoke:
    """Smoke tests that connect to Gemini 3.1 Flash Live and verify basic protocol."""

    def test_31_connects_and_gets_setup_complete(self, api_key):
        """Connect to 3.1, send setup, verify setupComplete response."""

        async def _run():
            if websockets is None:
                pytest.skip("websockets package required")

            ws_url = f"{GEMINI_WS_V1BETA}?key={api_key}"
            ssl_ctx = ssl.create_default_context()

            # Build a minimal 3.1 setup message
            func_decls = TOOL_DECLARATIONS[0]["functionDeclarations"]
            setup = {
                "setup": {
                    "model": f"models/{MODEL_31_FLASH_LIVE}",
                    "system_instruction": {"parts": [{"text": "You are a test assistant."}]},
                    "tools": [{"function_declarations": func_decls}],
                    "generation_config": {
                        "speech_config": {
                            "voice_config": {"prebuilt_voice_config": {"voice_name": VOICE}},
                        },
                        "response_modalities": ["AUDIO"],
                        "thinking_config": {"thinking_level": "minimal"},
                    },
                },
            }

            async with websockets.connect(ws_url, ssl=ssl_ctx, max_size=10 * 1024 * 1024) as ws:
                await ws.send(json.dumps(setup))

                # Wait for setupComplete
                deadline = asyncio.get_event_loop().time() + SETUP_TIMEOUT
                while True:
                    remaining = deadline - asyncio.get_event_loop().time()
                    if remaining <= 0:
                        raise TimeoutError("Timed out waiting for setupComplete from 3.1")
                    raw = await asyncio.wait_for(ws.recv(), timeout=remaining)
                    msg = json.loads(raw)
                    if "setupComplete" in msg or "setup_complete" in msg:
                        return  # success

        asyncio.run(_run())
```

```kotlin
fun sendTextPrompt(text: String) {
    if (ws == null || !setupDone) {
        Log.w(TAG, "sendTextPrompt skipped: ws=${ws != null}, setupDone=$setupDone")
        return
    }
    val msg = if (isV31) {
        // 3.1: text goes through realtimeInput
        buildJsonObject {
            putJsonObject("realtimeInput") {
                put("text", text)
            }
        }
    } else {
        // 2.5: text goes through client_content
        buildJsonObject {
            putJsonObject("client_content") {
                putJsonArray("turns") {
                    addJsonObject {
                        put("role", "user")
                        putJsonArray("parts") {
                            addJsonObject { put("text", text) }
                        }
                    }
                }
                put("turn_complete", true)
            }
        }
    }
    Log.d(TAG, "Sending text prompt: $text")
    ws?.send(msg.toString())
}
```

### Switching between versions

To switch the entire system between 2.5 and 3.1, change one line in `python/server.py`:

```python
# 3.1 (default):
GEMINI_LIVE_MODEL = "gemini-3.1-flash-live-preview"

# 2.5 (fallback):
GEMINI_LIVE_MODEL = "gemini-2.5-flash-native-audio-latest"
```

The Kotlin app reads this via `/api/config` and branches all protocol behavior automatically. The Python test harness uses its own `LIVE_MODEL` constant (update it manually to test the other version).
